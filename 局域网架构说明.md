# UpSnap 跨网段局域网自动化唤醒 (WOL) 技术手册

> [!IMPORTANT]
> 本文档是针对 UpSnap 在复杂网络环境（Proxmox LXC、NAT 隔离、跨网段、VLAN 物理隔离）下应用的技术全流程总结。

---

## 📜 第一部分：网络环境与架构布局 (Environment)

### 1. 物理隔离与逻辑网段
您的家庭网络采用了极硬核的“物理隔离、网络伪装、自动化中继”设计，旨在实现极致的安全与游戏防封：
*   **物理层 (Layer 1):** 混合式虚拟化平台。由 PVE 宿主机连接主路由器及交换机。利用交换机的“傻瓜 VLAN”模式实现物理端口间的硬隔离（1-14口完全隔离，15/16作为上联口）。
*   **逻辑层 (Layer 3):** 采用 **“单臂旁路双重网段”**：
    *   **主管理网 (`192.168.3.x/24`):** 用于 PVE 管理、LXC 管理及移动端控制。
    *   **隔离子网 (`10.100.0.x/24`):** Windows 生产/游戏机所在的独立网域，网关指向影子 LXC。

### 2. 核心组件：Alpine "守护者" (The Guardian LXC)
每个隔离设备对应一个 Alpine 轻量级容器，作为其在主网的“代理人”，承担以下核心使命：
*   **流量特征重塑:** 修改 TTL 为 128 (克隆 Windows 路由跳数) 及适配 TCP MSS，隐藏代理/中继特征。
*   **访问控制 (nftables):** 通过防火墙实现单向穿透准入，杜绝隔离设备反向扫描主网。
*   **WoL 精准中继 (Socat-Trigger):** 
    *   监听 UDP 9 端口。实现了 **“IP 即开关”**：主网软件向 LXC 的 IP 发送单播请求，LXC 在二层隔离网内转发真正的魔术包。
    *   **配置逻辑:** 当 LXC (`192.168.3.130`) 收到 UDP 9 命令，立即触发 `ether-wake` 给内部 Windows 物理网卡。

### 3. 精准唤醒 6 步流程
1.  **UpSnap 指令:** 用户点击启动，发出 **定向单播** UDP 包至影子 LXC IP。
2.  **防火墙校验:** LXC 的 `nftables` 验证源 IP (UpSnap) 并放行。
3.  **Socat 响应:** LXC 内部进程捕获指令，触发后台唤醒脚本。
4.  **魔术包生成:** LXC 执行 `ether-wake`，在特定网口发出包含 Windows MAC 地址的二层包。
5.  **交换机分发:** 交换机上联口收到广播包，根据 VLAN 逻辑分发至所有物理隔离口。
6.  **硬件拉起:** 目标 Windows 网卡识别 Magic Packet 信号，实现开机。

## ❌ 第二部分：UpSnap 原版核心故障回顾 (Problems)

在自动化维护上述架构时，UpSnap 原版代码暴露出如下深度 Bug：

1.  **Cron 任务“幽灵缓存”:** 当您在 UI 修改设备 IP 或 Ping 指令后，后端 Go 逻辑未触发重载。只要 Cron 表达式没变，它就会永远使用启动时的那一套旧参数。
2.  **闭包变量捕获 Bug:** 多设备高并发环境下，由于在迭代中错误使用了全局指针，导致定时任务触发时参数乱套，经常“指鹿为马”。
3.  **探测报错干扰 (Exit Status 1):** 当使用自定义 Ping 探测 Radmin 端口时，若机器离线，系统会简单粗暴地将正常的非零退出码判定为程序崩溃（ERROR），导致后台日志狂刷，无法分辨真实故障。
4.  **Wake 序列死锁:** 如果唤醒命令执行过快或启动失败，设备会永久锁死在 `pending` 状态，不再响应后续的任何自动唤醒请求。

## 🛠️ 第三部分：深度修复方案与步骤 (Solution Steps)

我们针对您的环境定制了 **UpSnap v1.0.8**，彻底消灭了上述问题：

### 1. 源码级核心修复 (Implemented in v1.0.8)
*   **配置热重载 (`pb.go`):** 增加了对设备核心网络字段的深度监控。现在修改 IP 或探测脚本会立即引发 Cron 销毁与重建，无需重启。
*   **作用域安全化:** 重构了 `cronjobs.go` 的协程逻辑，确保每个微任务执行时都拥有最新的、被锁定的独立设备内存结构。
*   **Ping 逻辑鲁棒化:** 
    *   **全退出码兼容:** 允许返回码 `2` (nc 超时) 等非零值代表正常的“离线”状态，不再作为 ERROR 处理。
    *   **日志静默化:** 只有当探测命令真正无法执行（找不到命令等）时才会记录错误，常规的离线检测将不再骚扰日志。
*   **Wake 进程守候:** 引入了 WaitGroup 理念，确保唤醒进程在任何情况下（成功、超时、崩溃）都能在结束时正常释放状态锁。

### 2. 自动化构建与快速部署
*   **容器镜像:** 迁至 **GHCR** 托管 (`ghcr.io/monstercjz/UpSnap`)。
*   **部署步骤:** 
    ```bash
    # 只需要执行这两步即可应用所有修复
    docker-compose pull 
    docker-compose up -d
    ```
